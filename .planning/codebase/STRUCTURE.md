## Directory Purposes

**src/:**
- Purpose: All TypeScript source files for the MCP server
- Contains: Core logic, types, utilities, entry point
- Key files: `mcp-server.ts` (main), `file-utils.ts`, `storage-utils.ts`, `file-watcher.ts`
- Build output: Compiled to `dist/` via esbuild

**dist/:**
- Purpose: Compiled JavaScript output ready for Node.js execution
- Generated by: `npm run build` (esbuild)
- Entry file: `dist/mcp-server.js` (runs the server)
- Not committed: Excluded via .gitignore

**.claude/:**
- Purpose: Claude-specific configuration and extensions
- Contains: Custom commands, agent definitions, hooks, GSD templates
- Not part of core server: Used by Claude Code integration

**.planning/codebase/:**
- Purpose: Architecture and code analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: GSD `/gsd:map-codebase` command
- Not committed: Excluded via .gitignore

**.github/workflows/:**
- Purpose: CI/CD automation
- Contains: Build and test workflows
- Triggers: Push, pull requests

**coverage/:**
- Purpose: Test coverage reports
- Generated by: `npm run coverage`
- Not committed: Excluded via .gitignore

## Key File Locations

**Entry Points:**
- `src/mcp-server.ts`: Main server file. Contains MCP server initialization, tool definitions, stdio transport, orchestration logic (lines 1102-1114 for startup)

**Configuration:**
- `config.json`: Base configuration with baseDirectory, excludePatterns, fileWatching settings
- `tsconfig.json`: TypeScript compiler options (ES2020, ESNext modules, strict mode)
- `vitest.config.ts`: Test framework configuration
- `package.json`: Dependencies (MCP SDK, chokidar, zod) and build/test scripts

**Core Logic:**
- `src/file-utils.ts`: File scanning, import parsing, dependency graph building, importance calculation (700+ lines)
- `src/storage-utils.ts`: Tree persistence, config management, path resolution
- `src/file-watcher.ts`: File system change detection via chokidar
- `src/global-state.ts`: Project root and config management
- `src/config-utils.ts`: Configuration loading and validation (Zod schemas)
- `src/types.ts`: All TypeScript type definitions and classes
- `src/logger.ts`: Logging utilities

**Testing:**
- `src/file-utils.test.ts`: Unit tests for path normalization and utility functions

## Naming Conventions

**Files:**
- TypeScript: `*.ts` (source) or `*.test.ts` (tests)
- Compiled JavaScript: `*.js` in `dist/`
- Configuration: `*.json`
- Pattern: kebab-case (e.g., `file-utils.ts`, `mcp-server.ts`)

**Directories:**
- All lowercase with hyphens for compound words
- Example: `.claude`, `.github`, `src/`, `dist/`

**Functions:**
- camelCase throughout (e.g., `buildFileTree()`, `normalizePath()`, `scanDirectory()`)
- Prefix async functions with verb: `async loadConfig()`, `async saveFileTree()`

**Classes:**
- PascalCase (e.g., `FileNode`, `AsyncMutex`, `StdioTransport`, `FileWatcher`)

**Constants:**
- UPPER_SNAKE_CASE (e.g., `DEFAULT_FILE_WATCHING`, `DEBOUNCE_DURATION_MS`, `INTEGRITY_SWEEP_INTERVAL_MS`)

**Types/Interfaces:**
- PascalCase (e.g., `FileTreeConfig`, `ToolResponse`, `FileWatchingConfig`)

## Where to Add New Code

**New Feature:**
- Primary code: `src/mcp-server.ts` for tool registration, `src/file-utils.ts` for analysis logic
- Tests: `src/file-utils.test.ts` or create new `src/feature-name.test.ts`
- Example: Adding a new ranking algorithm → update `calculateImportance()` in `file-utils.ts`, add tests

**New Tool Endpoint:**
- Location: `src/mcp-server.ts` (lines 639-1099 for tool definitions)
- Pattern: Use `server.tool("name", "description", {params...}, async handler)` with Zod validation
- Always guard with `if (!isProjectPathSet()) return projectPathNotSetError;`
- Example: `server.tool("new_feature", "...", {...}, async (params) => { ... })`

**New Helper Module:**
- Location: Create `src/new-feature.ts` with kebab-case naming
- Export functions for use by other modules
- Add types to `src/types.ts` if needed
- Example: File comparison logic → `src/file-compare.ts`

**New Data Types:**
- Location: `src/types.ts`
- Pattern: Class or interface with export keyword
- Example: New config option → add to `FileWatchingConfig` interface or `Config` interface

**Utilities/Helpers:**
- Shared path utilities: `src/file-utils.ts` (already has `normalizePath`, `toPlatformPath`)
- Storage utilities: `src/storage-utils.ts` (path resolution, file I/O)
- Global state access: `src/global-state.ts` (project root, config)

**Build Configuration:**
- TypeScript: `tsconfig.json` (already strict mode, ES2020)
- Build: `package.json` `build` script uses esbuild with specific entry files
- Add new source file: Update `build` script in `package.json` to include in esbuild arguments

## Special Directories

**node_modules/:**
- Purpose: NPM packages
- Generated: By `npm install`
- Committed: No (in .gitignore)
- Key dependencies: `@modelcontextprotocol/sdk`, `chokidar`, `zod`

**dist/:**
- Purpose: Compiled JavaScript ready to run
- Generated: By `npm run build` via esbuild
- Committed: No (in .gitignore)
- How to rebuild: `npm run build` or `./build.sh` (bash) / `build.bat` (Windows)

**coverage/:**
- Purpose: Test coverage reports
- Generated: By `npm run coverage`
- Committed: No (in .gitignore)
- View: `coverage/index.html` in browser

**FileScopeMCP-tree-*.json (generated):**
- Purpose: Cached file tree for a scanned project
- Created: By `buildFileTree()` when project is initialized
- Location: Project root or specified path
- Format: Serialized `FileTreeStorage` (config + tree)
- Committed: No (in .gitignore)

**FileScopeMCP-excludes.json (generated):**
- Purpose: Custom file/directory exclusion patterns per project
- Created: By `addExclusionPattern()` in global-state.ts
- Location: Project root
- Format: JSON array of glob patterns
- Committed: No (in .gitignore)
- Used by: `getConfig()` merges these with base excludePatterns

**mcp-debug.log (generated):**
- Purpose: Debug logging (optional)
- Created: By `log()` if `enableFileLogging(true)` is called
- Location: Current working directory
- Committed: No (in .gitignore)

## Build and Compilation

**Build Process:**
1. TypeScript source in `src/` with `.ts` extension
2. `npm run build` → esbuild transpiles to ES2020 JavaScript
3. Output: `dist/` directory with `.js` files matching source
4. Entry point compiled: `dist/mcp-server.js`

**Type Checking:**
- Run: `npm run typecheck`
- Config: `tsconfig.json` with `strict: true`

**Development Workflow:**
- Edit TypeScript in `src/`
- Run `npm run build` to compile
- Run `npm start` to execute compiled server
- Or use watch: `npm run test -- --watch` for tests

## Import/Export Patterns

**ES Modules:**
- All files use `import`/`export` syntax (ESNext)
- Type imports: `import type { FileNode } from './types.js'` (when available)
- Package imports: `import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js'`
- Relative imports: `import { normalizePath } from './file-utils.js'` (note `.js` extension even in TypeScript)

**Module Organization:**
- Circular dependency: Avoided by separating concerns (e.g., `global-state.ts` imported by many, imports nothing)
- Barrel files: Not used; direct imports preferred
