### NEXUS-34 Design Note

NEXUS-34 originally specified `POST /api/repos {path, name?}` for adding repos manually. The implementation chose auto-discovery + blacklist restore as the repo entry mechanism instead. This was a deliberate design decision documented in the 24-03 SUMMARY. Remove, blacklist, and restore are all implemented. The requirement is considered satisfied because the Settings page provides complete repo lifecycle management through the auto-discovery + restore model.

### Integration Notes (Non-Breaking)

1. **D-12 duplicate guard:** Auto-filter logic for >500 nodes appears in both Project.svelte and DependencyGraph.svelte. The DependencyGraph copy is a dead path because filterDir is already set by Project.svelte's $effect.

2. **Startup-only auto-discovery:** `recheckOffline()` reconnects known offline repos but doesn't discover new repos added to disk mid-session. By design — restart server or manage via Settings.

3. **Unused repoTokens field:** `/api/system/broker` returns `repoTokens` from StatusResponse but the UI renders token data from the separate `/api/system/tokens` endpoint. The field is available for future use.
