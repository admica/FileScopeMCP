# Requirements: FileScopeMCP

**Defined:** 2026-04-08
**Core Value:** LLMs get accurate, current answers about any file's role, relationships, and contents through MCP queries — without ever needing to read the raw files or maintain the metadata themselves.

## v1.4 Requirements

Requirements for Deep Graph Intelligence milestone. Each maps to roadmap phases.

### AST Extraction

- [ ] **AST-01**: LanguageConfig registry dispatches tree-sitter grammars per file extension via a single generic extractor
- [ ] **AST-02**: Tree-sitter extracts import edges for Python files (replacing regex)
- [ ] **AST-03**: Tree-sitter extracts import edges for Rust files (replacing regex)
- [ ] **AST-04**: Tree-sitter extracts import edges for C/C++ files (replacing regex)
- [ ] **AST-05**: Tree-sitter extracts import edges for Go files (replacing regex)
- [ ] **AST-06**: Regex fallback extracts import edges for unsupported languages (Zig, Lua, PHP, C#, Java)
- [ ] **AST-07**: Extractor produces richer edge types: imports, contains, inherits (for TS/JS)
- [ ] **AST-08**: Extractor produces edge weights (reference count between file pairs)

### Edge Metadata

- [ ] **EDGE-01**: Schema migration adds edge_type, confidence, confidence_score, and weight columns to dependencies table
- [ ] **EDGE-02**: Confidence constants module defines EXTRACTED (1.0) and INFERRED (0.8) tiers with numeric scores
- [ ] **EDGE-03**: All edges written by extractors carry confidence label and score
- [ ] **EDGE-04**: `get_file_summary` MCP tool returns edge types and confidence for dependencies

### Community Detection

- [ ] **COMM-01**: Louvain clustering groups files into communities using the dependency graph
- [ ] **COMM-02**: Community results cached with dirty-flag invalidation (recomputes on graph changes)
- [ ] **COMM-03**: `get_communities` MCP tool returns community members by file path (not integer IDs)
- [ ] **COMM-04**: Community membership stored in SQLite (`file_communities` table)

### MCP Polish

- [ ] **MCP-01**: Token budget parameter on list-returning MCP tools caps response size by item count
- [ ] **MCP-02**: Existing MCP tools surface new edge types and confidence in their responses

## Future Requirements

Deferred to future release. Tracked but not in current roadmap.

### Surprise Scoring

- **SURP-01**: Rank unexpected cross-module connections by composite surprise score
- **SURP-02**: Each surprising connection includes a `why` explanation

### Cascade Weighting

- **CASC-01**: Use confidence labels to weight staleness cascade priority (EXTRACTED > INFERRED)

### Nexus Graph Intelligence

- **NXGI-01**: Edge-type filtering in Cytoscape.js dependency graph
- **NXGI-02**: Community color coding on graph nodes
- **NXGI-03**: Community cluster layout mode

### Additional Languages

- **LANG-01**: Tree-sitter grammar for PHP (verify export shape first)
- **LANG-02**: Tree-sitter grammar for C# (when available)
- **LANG-03**: Tree-sitter grammar for Java
- **LANG-04**: Tree-sitter grammar for Zig (when stable npm package exists)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Cross-file call resolution | Requires type registry — HIGH complexity, defer to v1.5+ |
| Embedding-based similarity | Structured metadata serves LLM needs better (project decision) |
| Graph diff between scans | No clear use case; storing snapshots adds storage complexity |
| Surprise scoring | Speculative; Graphify-specific, not proven for file intelligence |
| Cascade weighting by confidence | Marginal benefit for added complexity; queue already sorts by importance |
| Real-time community updates | Louvain is batch-only; on-demand + dirty-flag cache is correct pattern |
| Nexus visualization updates | Defer until graph intelligence features are proven useful |
