1. **Extend src/logger.ts** with level-aware methods per RESEARCH.md pattern:
   - Refactor existing `log()` body into a shared `_write(prefix, message, args, toConsole, toDisk)` helper
   - `log()` calls `_write('', message, args, !daemonMode, logToFile)` (preserves existing behavior exactly)
   - Add `error(message, ...args)` — always to console AND disk: `_write('[ERROR]', message, args, true, true)`
   - Add `warn(message, ...args)` — same as log: `_write('[WARN]', message, args, !daemonMode, logToFile)`
   - Add `info(message, ...args)` — same as log: `_write('[INFO]', message, args, !daemonMode, logToFile)`
   - Add `debug(message, ...args)` — suppressed in daemon mode: `if (daemonMode) return; _write('[DEBUG]', message, args, true, logToFile)`
   - IMPORTANT: Do NOT replace the `console.error` inside `_write`/`log` — that is the output mechanism. Only replace console.error in the 4 application files.

   - Failed to normalize/save/load errors → `logger.error()`
   - Creating/saving/loading lifecycle events → `logger.info()`
   - Internal detail/no-op messages → `logger.debug()`
   - Import: `import { error, info, debug } from './logger.js';` (or `import * as logger from './logger.js'` — pick one pattern)

3. **Replace console.error in src/global-state.ts** per RESEARCH.md classification:
