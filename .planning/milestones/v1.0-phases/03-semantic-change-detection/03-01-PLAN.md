<objective>
Implement the AST parsing and semantic diff engine for TypeScript/JavaScript files using tree-sitter, plus the SQLite schema extension for exports snapshot storage.

Purpose: This is the core of Phase 3 — the ability to parse TS/JS files into structured export snapshots and compare snapshots to classify what semantically changed. This enables body-only changes to skip dependent re-evaluation (CHNG-04) and produces the typed SemanticChangeSummary interface that Phase 4 consumes (CHNG-02).

Output: Types, AST parser, semantic diff module, schema migration, repository functions, and tests.
</objective>

  <action>
    1. Create `src/change-detector/ast-parser.ts`:
       - Load tree-sitter via createRequire (same pattern as better-sqlite3 in src/db/db.ts):
         ```typescript
         import { createRequire } from 'module';
         const require = createRequire(import.meta.url);
         const Parser = require('tree-sitter') as typeof import('tree-sitter');
         const { typescript: TypeScriptLang, tsx: TSXLang } = require('tree-sitter-typescript');
         const JavaScriptLang = require('tree-sitter-javascript');
         ```
       - Create parser instances for each grammar: tsParser (TypeScript), tsxParser (TSX), jsParser (JavaScript)
       - Dispatch grammar by extension: .ts -> TypeScript, .tsx -> TSX, .js/.jsx -> JavaScript (per Pitfall 4 in RESEARCH.md)
       - Export `extractSnapshot(filePath: string, source: string): ExportSnapshot`:
         - Parse source with appropriate grammar
         - Use tree-sitter queries to extract exports:
           - Named exports: `(export_statement declaration: (_) @decl)` -> extract name from childForFieldName('name'), map node.type to ExportedSymbol.kind
           - Default exports: `(export_statement value: (_) @val)` -> kind='default', name from childForFieldName('name') or 'default'
           - Handle all declaration types: function_declaration, class_declaration, lexical_declaration, type_alias_declaration, interface_declaration, enum_declaration
           - For lexical_declaration (const/let), extract the variable name from the first variable_declarator child
           - Extract signature as the first line of the node text (declaration line, not body)
         - Use tree-sitter queries to extract import paths (replaces regex for TS/JS — CHNG-04):
           - `(import_statement source: (string (string_fragment) @path))`
           - `(export_statement source: (string (string_fragment) @path))` for re-exports
           - `(call_expression function: (identifier) @fn (#eq? @fn "require") arguments: (arguments (string (string_fragment) @path)))` for require()
           - `(call_expression function: (import) arguments: (arguments (string (string_fragment) @path)))` for dynamic import()
         - Return ExportSnapshot with filePath, exports array, imports array, capturedAt=Date.now()
       - If tree-sitter parsing throws, catch error, log warning, return null (caller handles as "unknown")
       - Export `isTreeSitterLanguage(ext: string): boolean` — returns true for .ts, .tsx, .js, .jsx

    2. Create `src/change-detector/semantic-diff.ts`:
       - Export `computeSemanticDiff(prev: ExportSnapshot | null, next: ExportSnapshot): SemanticChangeSummary`
       - If prev is null: return { changeType: 'unknown', affectsDependents: true, confidence: 'ast' } (first-parse case, Pitfall 6)
       - Compare prev.exports vs next.exports by name:
         - added = names in next but not prev
         - removed = names in prev but not next
         - changed = names in both but signature differs
       - Compare prev.imports vs next.imports (sorted JSON comparison)
       - If no exports differ AND no imports differ: return { changeType: 'body-only', affectsDependents: false }
       - If exports differ:
         - Check if ALL changed symbols are kind='type' or kind='interface': return 'types-changed'
         - Otherwise: return 'exports-changed'
         - Set changedExports to the list of added/removed/changed names
       - affectsDependents = true for exports-changed, types-changed, mixed, unknown; false for body-only, comments-only
       - All results get confidence: 'ast'

    3. Write comprehensive tests:
       - `src/change-detector/ast-parser.test.ts`: Test extractSnapshot with various TS/JS code snippets covering all export forms (named function, default class, const, type, interface, enum, re-export). Test import extraction. Test that string literals containing "import" are NOT extracted (CHNG-04 verification). Test .tsx dispatching.
       - `src/change-detector/semantic-diff.test.ts`: Test computeSemanticDiff with all scenarios from the behavior list above.

    IMPORTANT: tree-sitter query syntax may need adjustment based on the actual grammar node types. If a query fails at runtime, inspect the CST using `tree.rootNode.toString()` to see the actual node structure, and adjust queries accordingly. The research document's query patterns are best-effort and may need tweaking.
