  <action>
    Create `src/db/repository.ts` following RESEARCH.md Pattern 3 with these functions:

    1. `getFile(filePath: string): FileNode | null` — select from files where path = filePath, convert row to FileNode via helper.
    2. `upsertFile(node: FileNode): void` — insert with onConflictDoUpdate on path. Map FileNode fields to DB columns. Per RESEARCH.md anti-pattern: do NOT reconstruct the entire nested tree — work per-row.
    3. `deleteFile(filePath: string): void` — delete from files where path = filePath. Also delete from file_dependencies where source_path or target_path = filePath (cascade cleanup).
    4. `getChildren(dirPath: string): FileNode[]` — LIKE prefix query + post-filter for immediate children (one level deep). Follow RESEARCH.md Pattern 3 exactly.
    5. `getDependencies(filePath: string): string[]` — select target_path from file_dependencies where source_path = filePath.
    6. `getDependents(filePath: string): string[]` — select source_path from file_dependencies where target_path = filePath.
    7. `setDependencies(sourcePath: string, localDeps: string[], packageDeps: PackageDependency[]): void` — delete existing deps for sourcePath, insert new ones. Store PackageDependency.name, .version, .isDevDependency in the extra columns.
    8. `getAllFiles(): FileNode[]` — select all from files, map each row to FileNode.
    9. Internal helpers: `rowToFileNode(row)` and `fileNodeToRow(node)` for bidirectional conversion. `rowToFileNode` must reconstruct the `dependencies`, `dependents`, and `packageDependencies` arrays by querying file_dependencies. Cache-friendly: only populate these when explicitly needed (getFile populates them; getAllFiles skips them for performance — caller can request separately).

    IMPORTANT: Per RESEARCH.md anti-pattern, `dependents` is NEVER stored as rows — it is derived by querying `getDependents()`. The repository reconstructs the `dependents` array on `getFile()` calls for backward compatibility with existing code that reads `node.dependents`.

    Use `getDb()` from `./db.js` for all queries. All functions are synchronous (better-sqlite3 is sync).
