  <behavior>
    - getStaleness("/a.ts") returns { summaryStale: null, conceptsStale: null, changeImpactStale: null } for a fresh file
    - getStaleness("/a.ts") returns { summaryStale: 1000, conceptsStale: 1000, changeImpactStale: 1000 } after markStale(["/a.ts"], 1000)
    - getStaleness("/nonexistent.ts") returns all nulls (file not in DB)
    - get_file_summary response includes summaryStale field when file is stale
    - get_file_summary response does NOT include summaryStale when file is fresh
    - get_file_importance response includes all three stale fields when non-null
    - find_important_files response includes staleness per file entry
  </behavior>
  <action>
    1. Add `getStaleness(filePath: string)` to src/db/repository.ts:
       - Uses getSqlite() raw query: `SELECT summary_stale_since, concepts_stale_since, change_impact_stale_since FROM files WHERE path = ?`
       - Returns `{ summaryStale: number | null, conceptsStale: number | null, changeImpactStale: number | null }`
       - If no row found, return all nulls
       - Export the function

    2. Augment MCP tool handlers in src/mcp-server.ts:
       - Import getStaleness from repository
       - In `get_file_summary` handler (around line 375): after building response object, call getStaleness(normalizedPath). Spread non-null fields into response: `...(stale.summaryStale !== null && { summaryStale: stale.summaryStale })` etc.
       - In `get_file_importance` handler (around line 317): same pattern, spread all three stale fields if non-null
       - In `find_important_files` handler (around line 348): for each file in the mapped array, call getStaleness(file.path) and spread non-null fields
       - Do NOT modify `list_files` (returns full tree object, staleness is per-file not per-tree), `set_file_summary`, or `read_file_content` (not metadata queries)

    3. Create src/mcp-server.test.ts:
       - Test getStaleness directly against test DB
       - For MCP handler tests: if integration testing is too heavy (requires full server setup), test getStaleness + the response shape logic in isolation. The key contract is: non-null staleness fields appear in responses, null ones are omitted.
  </action>
