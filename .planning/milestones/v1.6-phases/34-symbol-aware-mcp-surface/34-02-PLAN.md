| LLM-agent → get_file_summary handler | Untrusted `filepath` string. Existing `normalizePath` hardening applies; no new threat surface from Phase 34. |
| find_symbol handler → findSymbols helper | In-process call; trusted boundary within the process. |
| get_file_summary handler → getDependentsWithImports / getSymbolsForFile | In-process call; trusted boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-06 | Tampering / Injection | find_symbol `name` parameter reaching raw SQL via findSymbols | mitigate | Plan 01 already binds name as a `?` parameter inside a parameterized SQL statement — no string interpolation. D-02/D-03 and T-34-01 from Plan 01 cover this. Plan 02's handler merely passes the validated `name` through; no new injection surface. |
| T-34-07 | Denial of Service | find_symbol flooded with huge `maxItems` | mitigate | Handler clamps `maxItems` to at most 500 via Math.min before calling findSymbols. User cannot bypass because the Zod schema accepts any integer but the handler clamps immediately. D-04 enforces. |
| T-34-08 | Denial of Service | find_symbol used with no match across many repeated calls | accept | Each call is a single indexed SQL lookup on `symbols(name)` plus a COUNT on the same WHERE clause — both fast. better-sqlite3 is synchronous and has no connection pool exhaustion risk at the scale this tool operates (one user, one MCP session). No rate limiting implemented; not required by ASVS L1 for a local developer tool. |
| T-34-09 | Information Disclosure | find_symbol with `exportedOnly: false` reveals non-exported (private) symbol names | accept | Intentional behavior per FIND-03. | T-34-10 | Information Disclosure | get_file_summary exposes import line numbers per dependent | accept | Intended behavior per SUM-02 and the tool description update (D-21). Allows agents to navigate directly to import statements. No cross-user isolation concern (single-user tool). |
