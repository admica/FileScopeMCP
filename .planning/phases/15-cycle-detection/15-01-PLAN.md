    Create `src/cycle-detection.ts` with three exported functions:

    1. `buildAdjacencyList(edges: Array<{source_path: string, target_path: string}>): Map<string, string[]>` — builds directed adjacency list from edge rows. Both source and target nodes are added to the map (targets get empty arrays if they have no outgoing edges). This ensures Tarjan's visits all graph nodes.

    2. `iterativeTarjanSCC(adj: Map<string, string[]>): string[][]` — the core algorithm. Uses an explicit work stack (array of `{node: string, neighborIdx: number}` frames) instead of recursion. Maintains `index` Map, `lowlink` Map, `onStack` Set, and `stack` array per standard Tarjan's. The work stack loop processes neighbors one at a time via `neighborIdx` continuation. When all neighbors are processed, pops the frame, updates parent lowlink, and checks if the node is an SCC root (`lowlink === index`). Returns ALL SCCs including singletons.

    3. `detectCycles(edges: Array<{source_path: string, target_path: string}>): string[][]` — the entry point. Calls `buildAdjacencyList`, then `iterativeTarjanSCC`, then filters: keep SCCs where `scc.length > 1` OR the single node has a self-edge (`adj.get(node)?.includes(node)`). Sort paths within each group alphabetically. Sort groups by their first element. Return the sorted result.

    /**
     * Returns all local_import dependency edges in one batch query.
     * Used by cycle detection to build the full dependency graph without N+1 queries.
     * Package imports are excluded — they are not actionable for cycle detection.
     */
    export function getAllLocalImportEdges(): Array<{ source_path: string; target_path: string }> {
      const sqlite = getSqlite();
      return sqlite
        .prepare(
          "SELECT source_path, target_path FROM file_dependencies WHERE dependency_type = 'local_import'"
        )
        .all() as Array<{ source_path: string; target_path: string }>;
    }
