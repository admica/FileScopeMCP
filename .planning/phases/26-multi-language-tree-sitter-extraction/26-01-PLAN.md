**Step 4: Create Python extractor function**

Add `extractPythonEdges` function after the Ruby extractor section. Follows the exact `(filePath, content, projectRoot) => Promise<EdgeResult[]>` signature.

AST node types to walk:
- `import_statement` — contains `dotted_name` or `aliased_import` named children. For `import os`, the named children are `dotted_name` nodes with `.text = "os"`. For `import os, json`, multiple `dotted_name` children.
- `import_from_statement` — has `module_name` field accessed via `node.childForFieldName('module_name')`. The module_name is a `dotted_name` or `relative_import` node.

Resolution logic (for parity with regex):
- Take the module name text (e.g., "os", "json", ".submodule")
- If it starts with `.` (relative): `path.resolve(path.dirname(filePath), moduleName)` then check file existence. `isPackage: false`.
- If it does NOT start with `.` (absolute): `isPackage: true`, `packageName: moduleName.split('.')[0]`.

Handle the Python grammar returning `null` parser: if `pythonParser` is null, fall back to the regex extractor for `.py`.

```typescript
async function extractPythonEdges(
  filePath: string,
  content: string,
  projectRoot: string
): Promise<EdgeResult[]> {
  const tree = pythonParser.parse(content);
  const edges: EdgeResult[] = [];

  function visitNode(node: any): void {
    if (node.type === 'import_statement') {
      // "import os" or "import os, json" — multiple dotted_name/aliased_import children
      for (let i = 0; i < node.namedChildCount; i++) {
        const child = node.namedChild(i);
        const moduleName = child.type === 'aliased_import'
          ? (child.childForFieldName('name')?.text ?? '')
          : child.text;
        if (moduleName) handlePythonModule(moduleName, filePath, projectRoot, edges);
      }
    } else if (node.type === 'import_from_statement') {
      const modNameNode = node.childForFieldName('module_name');
      if (modNameNode) {
        handlePythonModule(modNameNode.text, filePath, projectRoot, edges);
      }
    }
    for (let i = 0; i < node.childCount; i++) visitNode(node.child(i));
  }

  visitNode(tree.rootNode);
  return edges;
}
```

The `handlePythonModule` helper:
```typescript
async function handlePythonModule(
  moduleName: string,
  filePath: string,
  projectRoot: string,
  edges: EdgeResult[]
): Promise<void> {
  if (!moduleName || moduleName === '.') return;
  // Relative import: starts with '.'
  if (moduleName.startsWith('.')) {
    const resolved = path.resolve(path.dirname(filePath), moduleName);
    const normalized = normalizePath(resolved);
    try {
      await fsPromises.access(normalized);
      edges.push({
        target: normalized,
        edgeType: 'imports',
        confidence: EXTRACTED,
        confidenceSource: CONFIDENCE_SOURCE_EXTRACTED,
        weight: 1,
        isPackage: false,
      });
    } catch { /* file not found — skip */ }
  } else {
    // Absolute module: package dependency
    const topLevel = moduleName.split('.')[0];
    const resolved = path.resolve(path.dirname(filePath), moduleName);
    const normalized = normalizePath(resolved);
    edges.push({
      target: normalized,
      edgeType: 'imports',
      confidence: EXTRACTED,
      confidenceSource: CONFIDENCE_SOURCE_EXTRACTED,
      weight: 1,
      isPackage: true,
      packageName: topLevel,
    });
  }
}
```

**Step 5: Create Rust extractor function**

Add `extractRustEdges` function. AST node types:
- `use_declaration` — `childForFieldName('argument')` gives the use path text (e.g., "std::io", "crate::utils")
- `mod_item` — `childForFieldName('name')` gives the module name. ONLY emit edge if `!node.childForFieldName('body')` (bodyless = external reference; with body = inline module)
- `extern_crate_declaration` — `childForFieldName('name')` gives the crate name

Resolution logic:
- `crate::` prefix: relative to project root. Strip `crate::`, resolve: `path.join(projectRoot, 'src', restOfPath.replace(/::/g, '/'))`. Check file existence (try `.rs` extension). `isPackage: false`.
- `super::` prefix: relative to parent directory. `isPackage: false`.
- `self::` prefix: relative to current directory. `isPackage: false`.
- Everything else (e.g., `std::io`, `serde::Deserialize`): `isPackage: true`, `packageName: firstSegment` (before first `::`).

```typescript
async function extractRustEdges(
  filePath: string,
  content: string,
  projectRoot: string
): Promise<EdgeResult[]> {
  const tree = rustParser.parse(content);
  const edges: EdgeResult[] = [];

  function visitNode(node: any): void {
    if (node.type === 'use_declaration') {
      const argNode = node.childForFieldName('argument');
      if (argNode) handleRustUse(argNode.text, filePath, projectRoot, edges);
    } else if (node.type === 'mod_item') {
      const nameNode = node.childForFieldName('name');
      if (nameNode && !node.childForFieldName('body')) {
        handleRustMod(nameNode.text, filePath, projectRoot, edges);
      }
    } else if (node.type === 'extern_crate_declaration') {
      const nameNode = node.childForFieldName('name');
      if (nameNode) {
        edges.push({
          target: nameNode.text,
          edgeType: 'imports',
          confidence: EXTRACTED,
          confidenceSource: CONFIDENCE_SOURCE_EXTRACTED,
          weight: 1,
          isPackage: true,
          packageName: nameNode.text,
        });
      }
    }
    for (let i = 0; i < node.childCount; i++) visitNode(node.child(i));
  }

  visitNode(tree.rootNode);
  return edges;
}
```

Helper functions `handleRustUse` and `handleRustMod`:
- `handleRustUse(usePath, ...)`: parse `usePath` (e.g., "std::io"). If starts with `crate::`, `super::`, or `self::` treat as local. Otherwise `isPackage: true` with `packageName` = first segment before `::`.
- `handleRustMod(modName, ...)`: `mod utils;` means look for `utils.rs` or `utils/mod.rs` relative to the current file's directory. Check existence, push as local edge.

**Step 6: Create C/C++ extractor function**

Add `extractCEdges` function that accepts a parser parameter to handle both C and C++ grammars. AST node type: `preproc_include` with `path` field.

```typescript
function makeIncludeExtractor(parser: any) {
  return async function extractIncludeEdges(
    filePath: string,
    content: string,
    projectRoot: string
  ): Promise<EdgeResult[]> {
    const tree = parser.parse(content);
    const edges: EdgeResult[] = [];

    function visitNode(node: any): void {
      if (node.type === 'preproc_include') {
        const pathNode = node.childForFieldName('path');
        if (pathNode) {
          if (pathNode.type === 'system_lib_string') {
            // <stdio.h> — system include
            const name = (pathNode.text as string).slice(1, -1); // strip < >
            edges.push({
              target: name,
              edgeType: 'imports',
              confidence: EXTRACTED,
              confidenceSource: CONFIDENCE_SOURCE_EXTRACTED,
              weight: 1,
              isPackage: true,
              packageName: name,
            });
          } else if (pathNode.type === 'string_literal') {
            // "myfile.h" — local include
            const rawPath = (pathNode.text as string).slice(1, -1); // strip quotes
            const resolved = path.resolve(path.dirname(filePath), rawPath);
            const normalized = normalizePath(resolved);
            // Don't check file existence for C/C++ — parity with regex which also doesn't
            edges.push({
              target: normalized,
              edgeType: 'imports',
              confidence: EXTRACTED,
              confidenceSource: CONFIDENCE_SOURCE_EXTRACTED,
              weight: 1,
              isPackage: false,
            });
          }
        }
      }
      for (let i = 0; i < node.childCount; i++) visitNode(node.child(i));
    }

    visitNode(tree.rootNode);
    return edges;
  };
}
```

NOTE on C/C++ parity: The regex extractor for C/C++ uses `path.resolve(dirname, importPath)` for ALL matches, then checks `match[0].includes('<')` for package detection. The AST extractor uses `pathNode.type` to distinguish system vs local includes, which is MORE accurate. For local includes (`"header.h"`), parity requires: resolve path relative to file's directory, same as regex. For system includes (`<stdio.h>`), parity requires: target = the header name (same text captured by regex), `isPackage: true`. Critically, the regex extractor checks `access()` for local files and skips nonexistent ones; the C/C++ AST extractor should do the same for consistency — wrap the local include push in a try/catch with `fsPromises.access(normalized)`.
