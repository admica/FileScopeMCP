  <name>Task 2: Rate limiter, prompt templates, and schema extension</name>
  <files>src/llm/rate-limiter.ts, src/llm/prompts.ts, src/db/schema.ts, src/db/repository.ts, drizzle/meta/_journal.json, drizzle/0002_add_llm_columns.sql</files>
  <action>
1. Create `src/llm/rate-limiter.ts`:
   - Export `TokenBudgetGuard` class per RESEARCH.md Pattern 4
   - Constructor takes `{ maxTokensPerMinute?: number; tokenBudget?: number }` with defaults (40000/min, unlimited lifetime)
   - `canConsume(estimatedTokens: number): boolean` — checks sliding window AND lifetime
   - `recordActual(actualTokens: number): void` — increments counters; sets exhausted=true on lifetime limit breach
   - `recordError(): void` — no token increment, just logs
   - `isExhausted(): boolean` — getter for circuit breaker state
   - `reset(): void` — resets per-minute window (NOT lifetime); called externally if needed
   - `getLifetimeTokensUsed(): number` — getter for persistence in Plan 03
   - `setLifetimeTokensUsed(n: number): void` — setter for loading persisted state on startup
   - Private `rotateWindowIfNeeded()` checks if 60s elapsed since windowStart; if so resets windowTokens and windowStart

2. Create `src/llm/prompts.ts`:
   - Export `buildSummaryPrompt(filePath: string, fileContent: string): string` — instructs LLM to summarize purpose in 2-3 sentences; include file path for context
   - Export `buildConceptsPrompt(filePath: string, fileContent: string): string` — instructs LLM to extract functions, classes, interfaces, exports, purpose as structured JSON
   - Export `buildChangeImpactPrompt(filePath: string, diff: string): string` — instructs LLM to assess riskLevel, affectedAreas, breakingChanges, summary from a diff
   - Keep prompts concise to minimize token usage; include "respond in JSON" instruction for structured prompts

3. Extend `src/db/schema.ts`:
   - Add `concepts: text('concepts')` and `change_impact: text('change_impact')` columns to the `files` table (after `exports_snapshot`)
   - Add `llm_runtime_state` table with: `key TEXT PRIMARY KEY, value TEXT` (generic key-value for lifetime_tokens_used, budget_exhausted_at)

4. Create a new Drizzle migration for the schema changes:
   - Read existing `drizzle/meta/_journal.json` to find the next migration index
   - Create the SQL migration file using PRAGMA table_info() pattern from RESEARCH.md to check column existence before ALTER TABLE (for safety on older SQLite)
   - Also CREATE TABLE IF NOT EXISTS llm_runtime_state
   - Update the _journal.json to include the new migration entry

5. Extend `src/db/repository.ts`:
   - Export `dequeueNextJob(): { job_id, file_path, job_type, priority_tier, payload } | null` — SELECT from llm_jobs WHERE status='pending' ORDER BY priority_tier ASC, created_at ASC LIMIT 1; use raw getSqlite() prepared statement
   - Export `markJobInProgress(jobId: number): void` — UPDATE status='in_progress', started_at=Date.now()
   - Export `markJobDone(jobId: number): void` — UPDATE status='done', completed_at=Date.now()
   - Export `markJobFailed(jobId: number, errorMessage: string): void` — UPDATE status='failed', completed_at=Date.now(), error_message
   - Export `writeLlmResult(filePath: string, jobType: string, result: string): void` — UPDATE files SET summary=? (or concepts=? or change_impact=?) WHERE path=?; use switch on jobType
   - Export `clearStaleness(filePath: string, jobType: string): void` — UPDATE files SET {jobType}_stale_since = NULL WHERE path=?; map jobType to column name
   - Export `recoverOrphanedJobs(): number` — UPDATE llm_jobs SET status='pending', started_at=NULL WHERE status='in_progress'; return changes count
   - Export `loadLlmRuntimeState(key: string): string | null` and `saveLlmRuntimeState(key: string, value: string): void` for token budget persistence
   - All new functions use raw getSqlite() prepared statements (consistent with Phase 4 patterns)
