     - Private `async dequeueLoop(): Promise<void>`:
       - If stopped, return (exit loop)
       - Call `dequeueNextJob()`. If null: schedule next tick in POLL_INTERVAL_MS (5000ms), return

       - Call `markJobInProgress(job.job_id)`
       - try:
         - `const result = await this.runJob(job)`
         - `writeLlmResult(job.file_path, job.job_type, result.text)`
         - `clearStaleness(job.file_path, job.job_type)`
         - `markJobDone(job.job_id)`
         - `budgetGuard.recordActual(result.totalTokens)`
       - catch(err):
         - `markJobFailed(job.job_id, String(err))`
         - `budgetGuard.recordError()`
       - Schedule next tick in 500ms (fast drain when jobs available)

     - Private `async runJob(job): Promise<{ text: string; totalTokens: number }>`:
       - Switch on `job.job_type`:

           - Read file content: `const content = await fs.readFile(job.file_path, 'utf-8')` (wrap in try/catch; on ENOENT mark job failed with 'file_deleted', clear staleness, throw)
           - `const { text, usage } = await generateText({ model: this.model, prompt: buildSummaryPrompt(job.file_path, content), maxTokens: this.config.maxTokensPerCall ?? 1024 })`
           - Return `{ text: text.trim(), totalTokens: usage?.totalTokens ?? 0 }`
         - 'concepts':
           - Read file content same as above
           - `const { output, usage } = await generateText({ model: this.model, output: Output.object({ schema: ConceptsSchema }), prompt: buildConceptsPrompt(job.file_path, content), maxTokens: this.config.maxTokensPerCall ?? 1024 })`
           - Return `{ text: JSON.stringify(output), totalTokens: usage?.totalTokens ?? 0 }`

           - Return `{ text: JSON.stringify(output), totalTokens: usage?.totalTokens ?? 0 }`
           - Same Ollama JSON repair fallback
         - default: throw `Unknown job type: ${job.job_type}`

     - `BACKOFF_MS = 30_000` (30 seconds when rate limited)
