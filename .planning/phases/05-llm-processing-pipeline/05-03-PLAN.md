<task type="auto">
  <name>Task 1: Wire LLMPipeline into coordinator lifecycle</name>
  <files>src/coordinator.ts</files>
  <action>
1. Import at top of coordinator.ts:
   - `import { LLMPipeline } from './llm/pipeline.js';`
   - `import type { LLMConfig } from './llm/types.js';`
   - `import { loadLlmRuntimeState, saveLlmRuntimeState } from './db/repository.js';` (add to existing import)

2. Add private field to ServerCoordinator:
   - `private llmPipeline: LLMPipeline | null = null;`

3. In `init()` method, AFTER the database is opened and file tree is built (after `this._initialized = true`), add:
   ```
   // Start LLM pipeline if configured and enabled (non-blocking)
   const appConfig = getConfig();
   if (appConfig?.llm?.enabled) {
     this.startLlmPipeline(appConfig.llm);
   }
   ```
   This MUST be after DB open and after `_initialized = true` so the pipeline can access the database. Do NOT await -- pipeline starts asynchronously per RESEARCH.md anti-pattern 6.

4. Add private method `startLlmPipeline(llmConfig: LLMConfig): void`:
   - Create `new LLMPipeline(llmConfig)` and assign to `this.llmPipeline`
   - Load persisted budget state: `const savedTokens = loadLlmRuntimeState('lifetime_tokens_used');` If non-null, call `this.llmPipeline.getBudgetGuard().setLifetimeTokensUsed(parseInt(savedTokens, 10))`
   - Call `this.llmPipeline.start()`
   - Log "LLM pipeline started"

5. Add private method `stopLlmPipeline(): void`:
   - If `this.llmPipeline` is null, return
   - Persist budget state: `saveLlmRuntimeState('lifetime_tokens_used', String(this.llmPipeline.getBudgetGuard().getLifetimeTokensUsed()))`
   - Call `this.llmPipeline.stop()`
   - Set `this.llmPipeline = null`
   - Log "LLM pipeline stopped"

6. In `shutdown()`, BEFORE `closeDatabase()`, add:
   - `this.stopLlmPipeline();`

7. Add public methods for MCP tool access:
   - `toggleLlm(enabled: boolean): void`:
     - If enabled and pipeline not running: read config, call `startLlmPipeline(config.llm)`
     - If !enabled and pipeline running: call `stopLlmPipeline()`
   - `isLlmRunning(): boolean`: returns `this.llmPipeline?.isRunning() ?? false`

   ```typescript
   server.tool("toggle_llm", "Enable or disable the background LLM processing pipeline", {
     enabled: z.boolean().describe("true to start LLM pipeline, false to stop it"),
   }, async ({ enabled }) => {
     if (!coordinator.isInitialized()) {
       return { content: [{ type: "text", text: "Error: Project not initialized. Call set_project_path first." }], isError: true };
     }
     try {
       coordinator.toggleLlm(enabled);
       // Persist the toggle to config file so restart respects it
       const config = getConfig();
       if (config) {
         if (!config.llm) {
           config.llm = { enabled, provider: 'anthropic', model: 'claude-3-haiku-20240307' };
         } else {
           config.llm.enabled = enabled;
         }
         setConfig(config);
         await saveConfig(config);
       }
       return { content: [{ type: "text", text: `LLM pipeline ${enabled ? 'started' : 'stopped'}. Setting persisted to config.` }] };
     } catch (err) {
       return { content: [{ type: "text", text: `Error toggling LLM: ${err}` }], isError: true };
     }
   });
   ```

3. Import `saveConfig` from `./config-utils.js` at the top of mcp-server.ts if not already imported. Also ensure `getConfig` and `setConfig` are imported from `./global-state.js` (they likely already are).
